Privacy Policy

How Cyrus handles your data — last updated March 2026

Your data never leaves your device. Cyrus is a fully local application. No information you enter, generate, or store within this app is ever transmitted to, collected by, or stored on any server controlled by the developers of this software.

Data Stored Locally

Local Only

The following information is written to a SQLite database that lives entirely on your machine. Nothing listed here is sent anywhere.

  • Account credentials — your username and a bcrypt-hashed copy of your password. Your plaintext password is never stored.
  • Exchange API keys — your connect API key and secret. These are encrypted at rest using AES-256 and decrypted only when Cyrus needs to contact your exchange.
  • Exchange connections — the name, label, and status of each exchange account you add.
  • Automation rules — trigger conditions, action parameters, and execution logs for any custom commands you create.
  • Session tokens — short-lived JWT tokens kept in memory for the duration of your session; they are not persisted to disk.

Database location: %APPDATA%\Cyrus\Cyrus.db

Local-Only Backend

Localhost

Cyrus runs a lightweight Flask server bound exclusively to 127.0.0.1 (your machine's loopback address). This server is started automatically when you open the app and terminates when you close it. It is not accessible from the internet, your local network, or any other device — only the Electron window running on your machine can communicate with it.

Network Communications

Exchange APIs Only

The only outbound network requests Cyrus makes are direct calls to the official REST APIs of exchanges you have connected. These requests occur when you:

  • Load or refresh your open orders
  • Load or refresh your whitelisted withdrawal addresses
  • Validate a new API key connection in Profile
  • An automation rule fires and executes a trade action

All such requests are made using your own API keys, authenticated directly with the exchange. The Cyrus developers receive no copy of these requests or their responses.

API Key Security

Encrypted at Rest

Exchange API keys are sensitive credentials. Cyrus handles them with the following protections:

  • Keys are encrypted with AES-256 before being written to the local database.
  • Keys are decrypted in memory only at the moment a request to your exchange is made, then discarded.
  • Keys are never logged, displayed in plaintext after submission, or transmitted to any party other than your chosen exchange.
  • It is strongly recommended to create API keys with the minimum required permissions (read-only) and to restrict them to your IP address in your exchange's API settings.

No Analytics or Telemetry

Zero Tracking

Cyrus does not include any of the following:

  • Usage analytics or event tracking
  • Crash reporting or error telemetry sent to external services
  • Advertising identifiers or fingerprinting
  • Background syncing of any kind
  • Third-party SDKs that collect or transmit personal information

Your Control

Full Control

Because all data is local, you are always in complete control:

  • Delete a connection — removes that exchange's API keys from the database immediately via the Profile page.
  • Delete all data — uninstalling the application and deleting %APPDATA%\Cyrus removes everything permanently.
  • Offline use — Cyrus works entirely without an internet connection except when explicitly querying your exchanges.
